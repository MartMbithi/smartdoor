# SmartDoor IoT Access Monitoring System

## Overview

SmartDoor is an Internet of Things (IoT) based access monitoring system designed to demonstrate a complete end-to-end IoT architecture. The system replaces traditional door access mechanisms with a digitally monitored access control solution that records and analyzes access events in real time.

The system integrates an embedded edge device, cloud database infrastructure, and a modern data visualization dashboard to provide real-time monitoring and analytics of access activity.

The project demonstrates how IoT technologies can be used to improve security monitoring by capturing access events, transmitting them to a cloud platform, and presenting the information through an interactive monitoring interface.

---

## System Architecture

The SmartDoor system follows a layered IoT architecture consisting of five primary layers.

### 1. Sensing Layer

The sensing layer represents the physical interaction with the door access system.

Components include:

- Push buttons simulating authorized and unauthorized access
- LED indicators for access feedback
- Buzzer for audio alerts
- OLED display for user feedback

These components interact with the embedded microcontroller to generate access events.

---

### 2. Edge Processing Layer

The edge processing layer is implemented using an ESP32 microcontroller. This device handles event detection, local feedback, and communication with external systems.

Responsibilities of the edge layer include:

- Detecting access events
- Providing visual and audible feedback
- Generating structured event data
- Communicating with cloud services
- Time synchronization using NTP

Each access event generates a structured data record containing:

- Device identifier
- Access status (granted or denied)
- Timestamp

---

### 3. Communication Layer

The system uses WiFi connectivity to transmit access data to cloud services.

Two communication mechanisms are used:

**HTTP REST Communication**

Access events are sent to Firebase Realtime Database using HTTP POST requests.

**MQTT Messaging**

Access events are also published to an MQTT broker, enabling real-time message distribution and integration with additional monitoring systems.

---

### 4. Cloud Data Management Layer

The cloud layer uses Firebase Realtime Database to store access logs generated by the edge device.

Example record:

{
"device": "SmartDoorNode01",
"status": "granted",
"timestamp": "2026-03-08 22:13:28"
}

Firebase provides:

- scalable event storage
- real-time data availability
- simple REST API access

---

### 5. Application Layer

The application layer is implemented using a Streamlit dashboard that visualizes access data stored in Firebase.

Dashboard capabilities include:

- access log visualization
- access statistics
- real-time monitoring
- anomaly detection alerts

---

## Dashboard Features

### Login Authentication

Secure login page to restrict dashboard access.

### Access Summary

Displays real-time statistics of granted and denied access attempts.

### Access Timeline

Visualizes access events over time.

### Daily Access Statistics

Provides daily trends of system usage.

### Access Logs

Displays complete access history in a searchable table.

### Security Alert Detection

Generates alerts when multiple denied attempts occur within a short time window.

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/MartMbithi/smartdoor.git
cd smartdoor-iot
```

---

### 2. Create a Virtual Environment

```bash
python3 -m venv venv
```

---

### 3. Activate the Virtual Environment

#### Linux / macOS

```bash
source venv/bin/activate
```

#### Windows

```bash
venv\Scripts\activate
```

---

### 4. Install Dependencies

```bash
pip install -r requirements.txt
```

---

### 5. Run the Dashboard

```bash
streamlit run dashboard.py
```

---

### 6. Open the Dashboard

After running the command above, open your browser and navigate to:

```
http://localhost:8501
```
